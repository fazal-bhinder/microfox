# YouTube Data API v3 TypeScript SDK Summary

This document summarizes the YouTube Data API v3 for the purpose of generating a TypeScript SDK.  It covers various endpoints for interacting with YouTube data.

## General Information

* **Base URL:** `https://www.googleapis.com/youtube/v3`
* **Authentication:**  API Key or OAuth 2.0 token.  OAuth 2.0 is required for any insert, update, delete requests, and requests for private user data.
* **Request Headers:**
    * `Accept-Encoding: gzip` (for gzip compression)
    * `User-Agent: <your-application-name> (gzip)` (for gzip compression)
    * `Authorization: Bearer <oauth2-token>` (for OAuth 2.0)
    * `If-Match` and `If-None-Match` (ETag support - check client library compatibility)
* **Error Handling:**  Standard HTTP error codes are used.  Refer to the [YouTube Data API Errors](https://developers.google.com/youtube/v3/docs/errors) documentation.
* **Quota Usage:**  Each API call consumes quota units. See the [Quota costs for API requests](https://developers.google.com/youtube/v3/guides/implementation/quota) table.
* **Partial Responses:** Use the `part` parameter (required) to specify which resource properties to include, and the `fields` parameter to further filter nested properties. This optimizes bandwidth and performance.
* **ETags:** Use the `If-Match` and `If-None-Match` headers for caching and preventing accidental overwrites.
